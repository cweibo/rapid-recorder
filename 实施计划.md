# 录屏软件开发实施计划 (增加 GIF 支持)

本计划旨在开发一个基于 C++ 的高性能 Windows 录屏软件。为了确保极致的录制性能和低资源占用，我们将使用 Windows 原生技术栈，不依赖任何第三方库。

## 用户需求确认

> [!IMPORTANT]
> - **录制范围**：第一阶段仅支持**全屏录制**。
> - **音频**：**不录制**音频。
> - **帧率**：固定 **30 FPS**。
> - **输出内容**：支持 **MP4 (H.264)** 和 **GIF 动画** 两种格式。

## 技术方案

本项目将直接调用 Windows 底层 API 以获得最佳性能：
1. **画面捕获 (Capture)**：使用 **DirectX 11 Desktop Duplication API (DXGI)**。这是目前 Windows 上延迟最低、性能最强的抓屏方式。
2. **视频编码 (MP4)**：使用 **Windows Media Foundation (SinkWriter)**。利用显卡硬件加速进行实时 H.264 编码。
3. **GIF 编码 (GIF)**：使用 **Windows Imaging Component (WIC)**。WIC 支持多帧 GIF 编码，并允许我们设置帧延迟 (`Delay`) 和循环次数 (`LoopCount`)。
4. **用户界面 (UI)**：使用原生的 **Win32 API** 构建一个现代、简约的控制面板。

## 详细开发模块

### 1. 核心逻辑模块

#### `CaptureEngine.h/cpp` (画面捕获)
- 负责从 GPU 提取帧纹理。

#### `VideoEncoder.h/cpp` (MP4 视频编码)
- 负责将纹理送入 Media Foundation 进行 H.264 编码。

#### `GifEncoder.h/cpp` (GIF 动画编码)
- **目标**：将截取的帧序列保存为 `.gif`。
- **关键技术**：
    - `IWICBitmapEncoder` 选择 GIF 容器。
    - 为每一帧设置 `IWICMetadataQueryWriter` 来指定 `Delay` (30fps 约相当于 3 乘以 10ms)。
    - 设置全局元数据以实现无限循环播放。

#### `Recorder.h/cpp` (录制调度)
- **改进**：根据用户选择的格式，动态初始化对应的编码器。

### 2. UI 界面模块

#### `main.cpp`
- **入口函数**：初始化界面，增加输出格式选择（MP4 / GIF）。

### 3. 构建系统

#### `build.bat`
- **链接库**：`d3d11.lib`, `dxgi.lib`, `mfplat.lib`, `mfreadwrite.lib`, `windowscodecs.lib` 等。

## 验证计划

### 自动化验证
- 验证系统能否成功创建 GIF 编码器。

### 手动验证
- 录制一段 5 秒的动画并保存为 GIF，检查其是否能循环播放。
- 确认 MP4 视频在播放器中正常打开。
